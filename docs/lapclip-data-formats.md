# LapClip `result.php` データ書式メモ

パーサ（[`src/userscript/parseDom.ts`](../src/userscript/parseDom.ts)）が依存する、LapClip リザルトページの DOM 構造とレース種別ごとの書式メモです。**レース当日のパーサー校正**の基準として使います。

> ここに載せるのは「書式の構造」だけです。実際のリザルト（選手名・タイム）は保存・転記しません。例はすべて**書式を模した合成値**です。

## DOM 構造

各エントリは 1 つの `<a class="result" name="{bib}">`。中に 2 ブロック:

- `.nwb` スパン ×3: `["{順位}位", "No.{bib}", "[{TEAM}]{氏名}"]`
- `.nw` スパン ×3: `["{フェーズ}", "{タイム}", "+Top : {ギャップ}"]`

順位は `.nwb[0]`（例 `12位`）。チームコードは氏名先頭の `[TUK]` 等。ギャップは `+Top : ` を除いた残り。

## レース種別と書式

| 種別 | フェーズ | タイム | ギャップ |
|---|---|---|---|
| **個人TT** | `中間点`（途中）/ `FINISH` | `0:02:57.94`（H:MM:SS.**cc** 1/100秒） | `+0:01.18` / `0:00.00` |
| **クリテリウム** | `10周`（完走＝周回数）/ `4周`（リタイア） | `0:33:47`（H:MM:SS 秒） | `+1:55`（時間差）/ `-4周`（周回遅れ） |
| **ロード** | `FINISH` / `3/6周`・`4/6周 SP2`（リタイア＝完了周回/総周回＋通過点） | `2:39:06`（H:MM:SS 秒） | `+2:54` / `-2周` |

## 種別の自動判定（`detectRaceShape`）

優先順位:

1. `ctg=004`（大鹿）→ **team_tt**（URLヒント。チームと個人は内容だけでは区別しづらいため）。
2. 1/100秒タイム（`\d\.\d{2}`）または `中間点` フェーズ → **individual_tt**（TTは必ずサブ秒計測）。
3. `N周` / `X/Y周` フェーズ → **mass_start**。
4. 秒精度で同タイムが複数（集団ゴール）→ **mass_start**。
5. それ以外（空ページ等）→ **individual_tt**（安全側の既定。データ掲載後に再判定）。

## 重要な手がかり

- **完走 vs 周回遅れ（マススタート）**: ギャップが**時間**（`+M:SS`）なら先頭集団／完走、`-N周` なら周回遅れ。フェーズが `FINISH` か `N周` かに関わらず使える信頼シグナル。
- **着順**: マススタートは集団同タイムのため**タイムで順位を決められない**。公式の `位`（`.nwb[0]`）をそのまま採用する。
- **フェーズ文法**: `(\d+)/(\d+)周` = 完了周回/総周回。末尾 `SP\d`/`KOM` = 直近の通過計測点。`N周`（クリテリウム）= 完了周回数（総周回はページに無い）。
- **総周回**: ロードはリタイア行の `X/6周` から取得可能。クリテリウムは `/総` 表記が無く不明（先頭の最大周回数で代用）。
- **未掲載ステージ**: 結果が無いと「データがありません」の1行のみ。`parseEntries` は拾わない（空＝待機表示）。
